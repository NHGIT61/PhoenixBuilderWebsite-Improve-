FastBuilder Phoenix

简介

FastBuilder 是一个为Minecraft Bedrock Edition设计的建筑辅助程序，可以在多个平台使用。Phoenix版本专门为网易版租赁服设计。目前支持的功能有欧几里得几何图形 · ACME文件之 ·BDP(BlockDump，FastBuilder私有格式)· schematic（无方块NBT数据）文件之 建筑生成，以及图像绘制。

提示：FastBuilder Phoenix 是商业化（付费）软件。

  • 本文中除特殊说明之处外，所述之 FastBuilder 均代表 FastBuilder Phoenix (Alpha)版本。
    您的浏览器不支持ruby注音系统，所有ruby将以「形（音）」格式展示。

原理

FastBuilder 采用了全新的技术，不再受限于 Websocket，因此在速度，性能和稳定性等方面有了巨大提升，并且拥有极大的可扩展性[但因为网易服务器的算力分割，致使FastBuilder程序无法全速发包给网易服务器，所以速度大多上线为10000方块/s]。当前FastBuilder的核心基于Sandertv的Gophertunnel （Gophertunnel是开源的MIT协议）。

源代码开放

FastBuilder Phoenix客户端 完整代码已开源（包括验证系统和核心算法），源代码点击此处可见（GPL v3协议）。

但开放源代码并不意味着免费，您必须拥有FastBuilder账号才能使用它。

  • 2021-11-1的条款更新（对于开发者），您可以对开源的Alpha版本进行第三方修改（比如功能完善,bug修复），再发布进行商业行为。当然您也可以给FastBuilder 官方repo发送Pull request，您编写的代码将有机会被广大FastBuilder用户所运行。

基本概念（世界观）

FastBuilder Phoenix与其他程序的不同点在于，存在「客户端」与「服务端」概念。玩家正在运行的游戏与 FastBuilder Phoenix 同为客户端，而玩家所欲使用 FastBuilder 于之中的租赁服为服务端。

众所周知，客户端不一定都要在同一台设备上，所以即使您自己的游戏[由WangYi代理运行的Minecraft中国版]没有运行也能使用或（正常运行）FastBuilder。

由于使用命令行操作，需要用户具备如下基础：

  • 文件操作能力，能够理解路径和文件层级。
  • 简单的英语能力，能够认出 “Error[错误]” · “Permission denied[文件访问权限被拒绝]” · “Not found[服务器无响应]” 等系统提示之字眼并明白其含义。
  • 能够区分全角符号[，]与半角符号[,]。
  • (最好具有)在命令行界面输入[打字]并执行命令[换行键]的能力。

请确保您具备以上能力，如果您不具备这样的条件仍执意购买 FastBuilder 导致无法(不会)使用，开发组[代理]不承担任何责任，更不会提供帮助。

必要条件

  • 拥有 Fast-Builder用户中心 账户也称为F-B账户

  • 因设备而异：

    • 电脑端（Windows/Linux/macOS）：电脑需具备正常的网卡
  • 安卓(Android): 拥有Termux, 安装步骤见下文
    • iOS: 设备已越狱(Jailbroken)且您了解终端的使用方法。
  • 勤劳的双手和善于思考的大脑
  1. 登录 FastBuilder 用户中心
  2. 进入 Profile 部分，设置网易版用户名（游戏名）
  3. 设定您要使用Fast Builder的租赁服的号码（注意：该租赁服必须允许任意等级的玩家加入）

  6. 再次进入【Profile】，设置辅助用户的名称
  7. 如果使用Fast Builder时显示【您已经被禁止登录游戏，如有问题请联系官方客服】，则表明辅助用户被封禁，届时请重新进入用户中心【网易实名】部分，点击丢弃辅助用户

      使用指导

FastBuilder Phoenix是纯命令行程序，没有复杂的GUI，这使得程序本身非常易于使用。

启动步骤

  • Windows: 双击FastBuilder可执行文件(.exe)

  • Linux: ./fastbuilder

  • iOS: 进入终端，执行以下命令:

    fastbuilder

  • Android: 打开Termux， 执行以下命令:

    ./fastbuilder

初始化程序

如果不出意外，在您完成了上述步骤，并且看到了Fast Builder成功启动的字样。第一次启动会要求用户输入Fast Builder账号密码（密码不显示），输入即可。完成后，Fast Builder会要求用户输入租赁服号码(Rental Server Number)和租赁服密码（没有可忽略，不会显示）。如果没有报错退出，那么FastBuilder大概就成功启动了（第二次启动不会要求用户输入账号密码[相当于保存]）。接下来，将其挂在后台，进入网易租赁服，若看到辅助用户在线，那么FastBuilder就成功运行了。这个时候，请给予辅助用户OP权限。 请在控制台输入FastBuilder命令，而非在游戏中，另外，我们也不推荐使用get命令，因为它将所有者的信息传入服务器后端，并可能导致封禁。

FastBuilder命令解析

FastBuilder采用类似Linux Shell的命令系统（并非原版Minecraft的命令），只需直接在Termux内输入。如果您是Linux爱好者,那您将会很快掌握FastBuilder的命令。

注意实际上「#」并不能在FastBuilder命令中作注释，此处只是为了叙述方便。

# 例子：生成半径为5,方向朝y轴的圆,以下两条命令是等价的
round -r 5 -f y -h 1
round --radius 5 --facing y --height 1
生成器设置

成功初始化FastBuilder之后，我们将辅助用户（Bot）所在的维度称为一个【空间】，所有的操作都是在该空间中进行的。因此，如果您想在不同的【空间】中使用FastBuilder，您必须让Bot处于目标空间。

想要使用FastBuilder，必须设置【空间】的【原点】（结构会在原点构造），默认的原点在Bot进入游戏时所处的位置。

使用 get 命令可以修改原点设置为您当前所处的位置，下面是相关功能命令的详解。

FastBuilder Phoenix使用了协程，这意味这您可以一次执行多个任务，您可以使用task命令管理任务。

Task命令

task命令用于管理当前的【任务】（每个建筑进程都被抽象为一个任务）。每个task都有自己的runtime，您可以用一些功能命令设定任务的内部参数。task命令具有如下几个基本子命令（#后面为注释，解释了子命令的作用）：

tasktype <type:async|sync> # Task类型，sync不支持进度显示，一边计算一边发送数据，async则是完成计算后发送，支持速度显示
当使用async模式时，所有的构建指令都支持 -resume 百分比 参数，例如 bdump -p xx.bdx -resume 90 即为跳过前90%，只构建最后 10%
task list # 列出所有任务的ID，内容以及状态
task pause <taskID> # 暂停某个任务
task resume <taskID> # 恢复某个被暂停的任务
task break <taskID> # 销毁某个任务（不可恢复，它将在列表中消失）

task命令还可以设定任务执行延迟(Delay)以及发包方案(Delay Mode):

task setdelay # 设定某个任务的发包延迟。在continuous模式下单位为微秒； 在discrete模式下单位为秒
task setdelaymode   # 设定某个任务的发包方案，有3个可选参数
task setdelaythreshold   # 设定某个任务的阈值（最大方块集合），仅在 discrete 方案下有效
  • continuous（连续）：以一定速度发包，不会突变，比较稳定。
  • discrete（离散）：每经过一段固定时间发送大量数据包（最大数量不大于阈值），突变，不稳定，有概率造成服务器崩溃。
  • none（无延迟）：持续发送大量数据包，不会突变，非常不稳定，很大可能造成服务器崩溃。

三种方法各有优缺点，请斟酌使用：

  • 速度: continuous <= discrete < none (部分配置也可能导致continuous > discrete)
  • 稳定性: continuous > discrete > none
  • 平滑：continuous > none >= discrete
功能命令

  • 设置空间原点/终点(终点并不一定是必要的)：

    set x y z
setend x y z

  • 设置全局指令执行延迟和执行方案：

    delay mode  # 设定默认的发包方案
delay threshold  # 设定默认阈值（最大方块集合），仅在 discrete 方案下有效
delay set # 设定默认的发包延迟。在continuous模式下单位为微秒； 在discrete模式下单位为秒

  • 设置是否显示进度条（显示建筑的进度百分比，方块总数，瞬时速度等信息。默认为true）

    progress <value:bool>

  • 从FastBuilder用户中心退出登录

    logout

  • 重新选择语言偏好

    lang

  • 打开FastBuilder控制菜单

    menu
建筑生成命令

从schematic · bdx(bdump)或mcacblock（ACME导出的建筑文件）文件中加载建筑：

  • -p –path 文件路径

    schem -p 文件路径
acme -p 文件路径
bdump -p 文件路径
-p --path 文件路径，暂时不支持有空格或特殊字符[如■♦等]的文件名。
# 可选Flags:
# --excludecommands : 不导入命令方块中的命令。
# --invalidatecommands : 无效化处理导入的命令方块中的命令，如命令"say 123"会被处理为"|say 123"
# -S --strict: 如果文件未签名或未能验证签名则不进行构筑。
像素画[平面或立体的]生成

  • 将图片加载到空间：

    mapart -p <图片路径> -mapX [横向由几张地图构成] -mapZ [纵向由几张地图构成] -mapY [允许使用的高度]

图片将根据 mapX, mapZ 自动裁剪，并缩放到合适的尺寸
mapY不填写时生成的地图画为2D平面，可用颜色为 51 色
mapY填写时生成的地图画最高为mapY指定的高度,可用颜色为153色

示例：
mapart -p xxx.jpg //生成一张地图画(128*128)
mapart -p xxx.jpg -mapX 4 -mapZ 3 //生成一张横向4张，纵向3张地图，总计12张地图构成的地图画
mapart -p xxx.jpg -mapX 4 -mapZ 3 -mapY 40 // 更丰富的色彩，但是限制最大使用高度为 40 格
